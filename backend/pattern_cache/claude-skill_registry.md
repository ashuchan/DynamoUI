# claude-skill_registry

Module: `backend/skill_registry/`
Role: Skill YAML schema definition, discovery, validation, and in-memory entity/enum/pattern index. No DB access — pure in-memory registry built at startup.

## Key Classes

| Class | File | Purpose |
|---|---|---|
| `EntitySkill` | `models/skill.py` | Pydantic v2 model for `*.skill.yaml`. Owns fields, display config, permissions, FK references. |
| `EnumSkill` / `EnumValue` | `models/enum.py` | Pydantic v2 model for `*.enum.yaml`. `EnumValue.deprecated` excludes from active_options(). |
| `PatternFile` / `Pattern` | `models/pattern.py` | Pydantic v2 models for `*.patterns.yaml`. Pattern has triggers (NL phrases) + query_template. |
| `Mutation` / `MutationFile` | `models/mutation.py` | Pydantic v2 models for `*.mutations.yaml`. All mutations require confirmation (Phase 1 invariant). |
| `SkillRegistry` | `models/registry.py` | In-memory runtime registry. Maps entity/enum names → skill objects, FK graph. |
| `EnumRegistry` | `registry/enum_registry.py` | Focused enum lookup: `get()`, `by_group()`, `is_valid_value()`, `active_options()`. |
| `LLMProvider` (ABC) | `llm/provider.py` | Provider abstraction. `create_provider()` factory returns Anthropic or Google impl. |
| `QuerySynthesiser` | `llm/query_synthesiser.py` | NL → QueryPlan via LLM. Hashes user input before logging — never logs raw user input. |
| `PatternSeeder` | `llm/pattern_seeder.py` | Phase 2+ stub. Generates cross-entity patterns during scaffold. |
| `Scaffolder` | `scaffold/scaffolder.py` | Live DB schema → `*.skill.yaml` stub. Output header: `# Auto-generated by dynamoui scaffold.` |

## Discovery & Validation

`loader/yaml_loader.py` → `discover_all()`:
- Finds `*.skill.yaml`, `*.enum.yaml`, `*.patterns.yaml`, `*.mutations.yaml`
- Pattern files **must** start with `# skill_hash: <16-char-hash>` (enforced by `hasher.py`)
- Returns `DiscoveryResult` with errors list

`loader/validator.py` → `run_validation()`:
- 4 phases in order: Schema → Cross-ref → Semantic → Connectivity
- Shadowed trigger detection: similarity > 0.85 = warning (not error)
- Returns `ValidationResult`. Any error → app exits(1) at startup — **never bypass**

## Formatters

- `formatters/llm_formatter.py`: Plain-text output for LLM context (lower token count than JSON). Deprecated enums flagged.
- `formatters/ui_formatter.py`: JSON output for UI dropdowns, field metadata, mutation defs.

## CLI (`dynamoui` command)

Defined in `cli/validate.py` (Click). Three commands:
- `dynamoui validate [--file FILE] [--check-connectivity]`
- `dynamoui scaffold --table TABLE --output FILE`
- `dynamoui compile-patterns` — recomputes skill hashes for pattern files

## Settings

`config/settings.py` — all Pydantic BaseSettings, loaded from env:
- `skill_settings` (`DYNAMO_SKILL_*`)
- `pg_settings` (`DYNAMO_PG_*`)
- `cache_settings` (`DYNAMO_CACHE_*`)
- `llm_settings` (`DYNAMO_LLM_*`)
- `internal_settings` (`DYNAMO_INTERNAL_*`)

All secrets are `SecretStr` — never hardcode, never log `.get_secret_value()` output.

## Critical Rules

- **Pydantic v2 only**: use `model_validator(mode='after')` and `field_validator`. Never use `@validator`.
- **Validation gate is sacred**: startup exits(1) on any validation error. Never soften or skip.
- **User input is never logged raw**: `QuerySynthesiser` hashes it before any log statement.
- **LLM default models**: Anthropic = `claude-haiku-4-5-20251001`, Google = `gemini-1.5-flash`.
